## 🔍 Проверка работы Битрикс24

### Шаг 1: Проверка подключения
Откройте консоль браузера (F12) и выполните:
```javascript
// Проверка конфигурации
console.log('Bitrix Config:', window.bitrixConfig);

// Проверка webhook
if (window.bitrixConfig.webhookUrl) {
    console.log('✅ Webhook URL найден');
    // Тестовый запрос
    window.testBitrixConnection();
} else {
    console.log('❌ Webhook URL не настроен');
}
```

### Шаг 2: Настройка в интерфейсе
1. Откройте **Настройки ОС** → вкладка **CRM (Bitrix24)**
2. Вставьте **Webhook URL** из Битрикс24
3. Нажмите **"Тест"** — должно показать "Связь успешно установлена!"
4. Выберите **Воронку продаж** и **Стадию производства**
5. Нажмите **"Сохранить"**

### Шаг 3: Проверка синхронизации
В **Менеджере заказов** (orders.html):
- Нажмите кнопку **↻** (синхронизация) в правом верхнем углу
- Проверьте статистику **"CRM Связь"** — должно показать % связанных сделок

| Функция | Описание | Где используется |
|---------|----------|------------------|
| `searchBitrixClient()` | Поиск клиента по имени/телефону | Калькулятор, Менеджер заказов |

| `updateBitrixDealSum()` | Обновление суммы сделки | При изменении заказа |

| `callBitrix()` | Универсальный вызов REST API | Все интеграции |

### 🔗 Автоматическая синхронизация

**При создании заказа в калькуляторе**:
1. Клиент ищется/создаётся в Битрикс24
2. Создаётся сделка с привязкой к контакту
3. Сумма и товары передаются в сделку
4. `bitrixDealId` сохраняется в заказе

**При изменении статуса заказа**:
1. Сделка перемещается на соответствующую стадию
2. Добавляется комментарий в ленту
3. Создаётся задача для руководителя (если настроено)

**При открытии карточки заказа**:
1. Загружается лента активности из Битрикс24
2. Показываются комментарии и задачи
3. Данные клиента подтягиваются из CRM

## 🚨 Возможные проблемы и решения

### Проблема 1: "Ошибка подключения к Битрикс24"
**Причина**: Неверный Webhook URL или он истёк  
**Решение**:
1. Зайдите в Битрикс24 → Приложения → Входящий вебхук
2. Создайте новый вебхук с правами:
   - `crm` (полный доступ)
   - `tasks` (задачи)
   - `user` (пользователи)
3. Скопируйте URL и вставьте в настройки ПРУТКОН ОС
4. Нажмите "Тест"

### Проблема 2: "Нет пользователей CRM"
**Причина**: Вебхук не имеет прав на чтение пользователей  
**Решение**: Проверьте права вебхука в Битрикс24

### Проблема 3: Сделка не создаётся
**Причина**: Не выбрана воронка или стадия в настройках  
**Решение**: 
1. Настройки → CRM → Выберите воронку
2. Выберите стадию "Новая сделка" и стадию "Производство"
3. Сохраните

### Проблема 4: Консоль показывает `null` вместо результата
**Причина**: Ошибка CORS или блокировка запроса  
**Решение**: 
- Проверьте консоль на ошибки сети
- Убедитесь что домен Битрикс24 не блокируется фаерволом

## 🧪 Тестовый сценарий

### 1. Создание заказа с интеграцией
1. Откройте **Калькулятор КП**
2. В поле **"Поиск по CRM"** введите имя клиента (мин. 3 символа)
3. Выберите найденного клиента или создайте нового
4. Добавьте товары в корзину
5. Нажмите **"Создать Сделку в Bitrix24"**
6. Проверьте в Битрикс24 — сделка должна появиться

### 2. Изменение статуса
1. Откройте **Менеджер заказов**
2. Найдите созданный заказ
3. Откройте карточку (стрелка вправо)
4. Нажмите **"Подтвердить заказ"**
5. Проверьте в Битрикс24 — стадия сделки должна измениться

### 3. Просмотр ленты активности
1. В карточке заказа прокрутите вниз
2. Должна отображаться лента из Битрикс24:
   - Комментарии
   - Задачи
   - Изменения статуса
