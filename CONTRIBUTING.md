# Правила участия в разработке ПРУТКОН ОС

Мы рады любому вкладу в развитие ПРУТКОН ОС. Чтобы процесс совместной работы был 
эффективным, пожалуйста, следуйте этим правилам.

## Как внести вклад

### Сообщение об ошибках
* Прежде чем открывать новый вопрос, проверьте существующие тикеты в журнале аудита.
* Опишите шаги, необходимые для воспроизведения ошибки.
* Укажите ожидаемое и фактическое поведение.

### Предложения по улучшению
* Опишите предлагаемую функцию и то, почему она была бы полезна для производства.
* Будьте готовы обсудить детали реализации с ведущим программистом.

## Стандарты кодирования
* Следуйте установленному стилю Avanhard Industrial: используйте чистый CSS с переменными и семантический HTML.
* Весь новый JavaScript код должен быть модульным и интегрированным через `core.js`.
* Обязательно обновляйте версию в `core.js` и фиксируйте изменения в `VERSION_HISTORY`.

## Процесс Pull Request
* Убедитесь, что ваш код не ломает существующую синхронизацию с Firebase.
* Сопроводите PR кратким описанием того, что было изменено.
* Ожидайте код-ревью перед слиянием с основной веткой.
